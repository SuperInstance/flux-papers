# FLUX Research Papers & Specifications

Academic and standards-track documentation for the **FLUX constraint system** — a formal framework for expressing, verifying, and enforcing temporal and spatial constraints in safety-critical systems.

## Contents

### Papers (`papers/`)
- **EMSOFT submission** — Final and draft versions of the EMSOFT paper on FLUX
- **Related work comparison** — Survey of constraint systems and formal methods

### Specifications (`specs/`)
- **Safe-TOPS/W benchmark** — Versions 1–4 of the Safe-TOPS/W metric specification
- **GUARD language** — Grammar, EBNF, conflict examples, and language spec
- **FLUX opcode reference** — Complete opcode table and C opcode reference
- **Temporal opcodes** — Time-based constraint opcodes
- **Security primitives** — Constraint-based security model
- **SDK CLI design** — Command-line interface specification
- **Constraint AST** — Universal constraint abstract syntax tree
- **FluxProject template** — Project configuration format
- **ISA integration spec** — Integration with instruction set architectures
- **Constraint check bridge** — Bridge between FLUX and verification tools

### Benchmarks (`benchmarks/`)
- ASCII benchmark reference for Safe-TOPS/W evaluation

### Architecture (`architecture/`)
- System architecture diagram and overview

### Man Pages (`man/`)
- `fluxc(1)` — FLUX compiler manual page

### Manifesto (`manifesto.md`)
- The FLUX project manifesto — motivation, vision, and design philosophy
